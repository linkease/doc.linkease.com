## 最常见问题

### ❓ 控制台显示设备离线

**可能原因：**

1. **刚操作了切换服务器、编辑域名、更换套餐**
   - 需要等待 1 分钟插件重新初始化
   - 如果 5 分钟后仍未在线，继续排查

2. **服务器升级维护**
   - 查看公告确认是否处于维护时段

3. **插件异常**
   - 检查 Token 是否填写正确
   - 确认设备能正常访问网络（ping ddnsto.com）
   - 确认令牌对应的微信账号是否正确

**排查步骤：**

```
1. 检查插件是否正常运行
   - 令牌是否填写正确
   - 如已离线，请重新运行
   - 如插件版本过低，请升级至最新版本

2. 确认设备能正常访问网络
   - ping ddnsto.com
   - ping kooldns.cn

3. 确认设备时间是否正确
   - 时间不正确会导致连接失败
   - 同步时间后等待一分钟或重启插件
```

---

### ❓ 访问域名失败

**排查清单：**

- [ ] 确认插件是否在线
- [ ] 确认插件所在设备与目标 IP 是否在同一网段
- [ ] 确认目标地址的端口是否正确
- [ ] 尝试访问该设备下其他域名
- [ ] 如刚编辑域名，请等待 1 分钟后尝试

**常见错误：**

| 错误提示 | 原因 | 解决方法 |
|---------|------|---------|
| `Forbidden Rejected request from RFC1918 IP` | 使用了 127.0.0.1 | 改用局域网 IP，如 `http://192.168.1.100` |
| 页面无法打开 | 目标服务未运行 | 检查内网服务是否正常 |
| 连接超时 | 端口错误或防火墙 | 检查端口和防火墙设置 |

---

### ❓ 远程 Windows 桌面失败

**排查步骤：**

1. **确认电脑已开启 RDP 功能**
   - Windows 家庭版需要使用 [RDPWrapper](https://github.com/stascorp/rdpwrap)
   - 专业版需要取消勾选"需要计算机使用网络级别身份验证"

2. **内网测试**
   - 先保证不使用 DDNSTO 的情况下，内网能够正常 RDP 访问

3. **确认从控制台入口进入**
   - 必须登录 DDNSTO 控制台，从远程应用入口进入

---

### ❓ 访问 OpenWrt 页面异常

部分使用 nginx 的固件通过 DDNSTO 访问时界面可能出现菜单不显示等问题。

**解决方法：**

1. 使用 HTTPS 访问
2. 手动加上路径 `/cgi-bin/luci/admin/status/overview`：
   ```
   https://custom.ddnsto.com/cgi-bin/luci/admin/status/overview
   ```

## 自助排查工具

### 一键收集日志

当需要技术支持时，请收集以下信息：

1. **设备信息**
   - 设备类型（路由器型号/NAS 型号）
   - DDNSTO 版本号
   - 插件运行状态截图

2. **网络信息**
   - 能否 ping 通 ddnsto.com
   - 能否 ping 通 kooldns.cn
   - 设备外网 IP（可在控制台查看）

3. **配置信息**
   - Token 前 5 位（不要发完整的！）
   - 域名前缀
   - 目标主机配置

4. **问题描述**
   - 问题发生时间
   - 问题现象截图
   - 已尝试的解决方法

## 快速修复清单

遇到问题时，按以下顺序尝试：

- [ ] 等待 1-2 分钟后刷新页面
- [ ] 检查 Token 是否填写正确
- [ ] 重启 DDNSTO 插件
- [ ] 检查设备网络连接
- [ ] 同步设备时间
- [ ] 升级 DDNSTO 到最新版本
- [ ] 清除浏览器缓存
- [ ] 更换浏览器尝试
