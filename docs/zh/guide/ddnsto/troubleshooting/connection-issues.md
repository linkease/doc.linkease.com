# 连接问题排查

> 解决设备离线、无法连接等网络相关问题

---

## 设备显示离线？四步自查

- 设备显示离线，不一定代表 DDNSTO 服务器异常。 很多时候是设备本身、网络环境、Token(令牌)、插件状态或系统时间等原因导致客户端没有正常连上平台。 如果你遇到这个问题，可以先按下面步骤自查。
- 检查插件版本是否过旧，是否升级到最新版本。

### 症状
- 控制台中设备状态显示"离线"
- 域名无法访问
- 之前正常，突然离线

### 排查步骤

#### 1. 检查 Token 配置

**问题：** Token 填写错误或过期

**检查方法：**
- 登录 [DDNSTO 控制台](https://www.ddnsto.com/app/#/login) 
- 查看令牌页面，确认 Token 是否正确
- 对比设备上填写的 Token 是否一致

**解决：**
- 关闭代理
- 重新复制 Token 填入设备
- 注意：Token 不要有多余空格

---

#### 2. 确认设备本身能正常联网 

- 请先确认你的路由器、NAS、iStoreOS、OpenWrt 或其他运行 DDNSTO 的设备是否可以正常上网。 可以在设备终端里执行： `date ping -c 4 223.5.5.5 ping -c 4 www.baidu.com curl -I https://www.ddnsto.com` 重点看这几项：

- `date` 显示的系统时间是否正确
- `ping 223.5.5.5` 是否正常，判断基础网络是否通
- `ping www.baidu.com` 是否正常，判断 DNS 是否正常
- `curl -I https://www.ddnsto.com` 是否能访问 DDNSTO 官网 如果 IP 能 ping 通，但域名 ping 不通，通常是 DNS 问题。 如果系统时间明显不对，可能会影响 HTTPS 连接，也可能导致设备无法正常连接平台。

---

#### 3. 确认 DDNSTO 进程是否还在运行 

- 如果设备能上网，但控制台仍然显示离线，请检查 DDNSTO 客户端或插件是否还在运行。 在设备终端执行： ps -ef | grep ddnsto | grep -v grep 如果没有任何输出，说明 DDNSTO 进程可能没有运行。 这时可以进入设备后台，检查 DDNSTO 插件状态，必要时重启插件或重启设备。

---

#### 4. 查看本机状态接口 

- 如果 DDNSTO 进程存在，可以继续查看本机状态接口： `curl -s http://127.0.0.1:18333/healthz curl -s http://127.0.0.1:18333/status` 重点看 /status 里的 connected 字段：

- 如果 connected=true，说明客户端已经连上 DDNSTO 平台，但控制台仍显示离线，可能是平台在线状态识别链路需要进一步分析。

- 如果 connected=false，通常表示客户端没有连上平台，常见原因包括 Token 错误、网络异常、防火墙拦截、系统时间异常等。

---

#### 5. 如果人在外面，设备不在身边怎么办？ 

- 如果现在不在设备旁边，但设备还有其他远程管理方式，可以尝试远程进入设备后台。如果装了我们易有云，可以尝试远程控制。 如果设备本身已经断网、路由器异常、NAS 关机，或者本地网络不可达，那么只能等本地网络恢复，或请现场人员协助重启设备/路由器。

---

## 刚操作后设备离线

### 症状
- 切换服务器后设备离线
- 编辑域名后设备离线
- 更换套餐后设备离线

### 原因
这是正常现象，插件需要重新初始化连接。

### 解决
1. 等待 1 分钟
2. 刷新控制台页面
3. 如果 5 分钟后仍离线，按上述排查步骤检查

---

## 服务器维护

### 症状
- 所有设备同时离线
- 控制台无法访问

### 原因
DDNSTO 服务器升级维护。

### 解决
1. 查看官方公告
2. 等待维护完成
3. 维护完成后设备会自动上线

---

## 网络白名单设置

### 症状
- 网络正常，但设备无法连接 DDNSTO
- 其他设备可以，特定设备不行

### 原因
路由器或防火墙拦截了 DDNSTO 连接。

### 解决
将以下地址添加到白名单：

```
tunnel.ddnsto.com
tunnel.kooldns.cn
tunnel.tocmcc.cn
www.kooldns.cn
www.tocmcc.cn
www.ddnsto.com
```

## 日志收集

当需要技术支持时，请提供以下日志：

### OpenWrt/iStoreOS

```bash
# 查看 DDNSTO 日志
logread | grep ddnsto

# 查看进程状态
ps | grep ddnsto
```

### Docker

```bash
# 查看容器日志
docker logs ddnsto
```

### 群晖

1. 打开套件中心 → DDNSTO → 查看日志
2. 或 SSH 登录后查看：`/var/log/ddnsto.log`

## 快速修复流程

```
1. 等待 1-2 分钟
   ↓
2. 刷新控制台页面
   ↓
3. 检查 Token 是否正确
   ↓
4. 重启 DDNSTO 插件
   ↓
5. 检查网络连接（ping ddnsto.com）
   ↓
6. 同步设备时间
   ↓
7. 升级 DDNSTO 版本
   ↓
8. 联系技术支持
```
